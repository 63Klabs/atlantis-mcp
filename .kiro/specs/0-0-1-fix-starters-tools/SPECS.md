# Fix Starters Tools

The starter tools are currently not returning results.

- mcp/list_starters
- mcp/list_starter_info

Let's rethink the way these tools are working.

We will remove access to GitHub. Similar to templates, all starters are available in S3.

We will continue to support multiple s3 buckets ["63klabs"] by default.

Starters are stored in S3 at the following location:

`{namespace}/app-starters/v2/{starter-name}.zip`

Anything that is a zip is repository contents in a single zip.
Anything that is a json is a side car file with information for that starter.

`{namespace}/app-starters/v2/{starter-name}.zip`
`{namespace}/app-starters/v2/{starter-name}.json`

If there is a zip without a json, that is okay.
If there is a json without a zip, we ignore the json file.

The json sidecar format will be as follows:

```json
{
    "name": "",
    "description": "",
    "languages": [],
    "topics": [],
    "frameworks": [],
    "dependencies": [],
    "devDependencies": [],
    "hasCacheData": true,
    "deployment_platform": "atlantis",
    "features": [],
    "prerequisites": [],
    "author": "",
    "license": "UNLICENSED",
    "repository": "github.com/{user}/{repo}",
    "repository_type": "app-starter",
    "version": "1.0.0",
    "last_updated": "Z"
}
```

This json is generated by the `scripts/generate-sidecard-metadata.py` script that will be invoked when the repository is packaged and deployed to S3 as a zip. This runs in a CI/CD pipeline. Make sure `generate-sidecard-metadata.py` will work within github actions or codebuild and is up to date with the specification above. features and pre-req should be provided by the README.md in the repo.

We should allow filtering by bucket and namespace.

We need to follow the same controller, service, fetchFunction, and models as templates.

Templates is fully implemented, you can use that as a resource. Pay particular attention to the service in the way it uses CachableDataAccess, ApiResponse, and the cacheObj.getBody(true);

Review documentation and tests to determine if anything else needs to be updated.
